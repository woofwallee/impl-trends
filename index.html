How to use

Documentation and reference for this dashboard

What this is
HubSpot shows you today. This dashboard shows you the trend. Import your monthly export and see how open implementations, stage times, and delivery speed are moving, and whether they're getting better or worse. The app opens with sample data so you can explore first. Your first real import replaces it.

The monthly routine
1. Export all Implementation records from HubSpot. 2. Import the CSV here. 3. That's it. The dashboard updates and remembers that month, so the picture builds over time.

Getting the export right
In HubSpot: CRM → Implementations → All records → Export. Choose CSV, include "All properties on records", and don't filter anything out. Open and closed both matter. Name the file implementations_YYYY-MM-DD.csv so the app knows which month it belongs to. The full field list is on the Import screen.

Reading the dashboard
The four numbers up top read at the end of your selected period, and each compares itself to the period before. Open Pipeline Trend shows whether open work is piling up or clearing. Time in Stage shows where work gets stuck. Click any stage to chart it. PO to Go-Live shows how long customers wait from Purchase Order to Go-Live. Go-Lives counts what actually shipped. The fourth number up top, Live, pending close, counts implementations that went live but haven't been moved to the Live/Complete stage in HubSpot yet. That's cleanup work waiting on someone.

What the colors mean
Green means improving. Red means worsening. That's the whole rule. The dotted line on each chart marks where the visible period started. Green below that line means better than the start, red above it means worse. Go-Lives bars stay neutral blue because they're a simple count. Its change pill turns green when more went live than the period before.

Timeframes and comparisons
Pick a preset (1M, 3M, 6M, 1Y, All) or set exact dates. Today snaps the end date to today. The product buttons (All, CAREpoint, e-Bridge) filter the whole dashboard to one product. Every number, chart, and comparison recalculates for that period. Change pills always compare to the equal-length period right before yours. One thing to know: charts open zoomed to the most recent two weeks, while the trend pill describes the whole selected period. So the pill can say "Rising" while the zoomed view shows a recent dip. Zoom out and they'll agree.

Exploring a chart
Scroll to zoom. Drag to pan. Double-click to reset. The floating bar at the bottom of each big chart does the same with buttons: zoom in and out, step back and forward through time, reset. On Time in Stage, the ‹ handle tucks the stage list away so the chart gets the full width. Press it again to bring the list back.

Why product counts can add past the total
An implementation can be both CAREpoint and e-Bridge. It counts once in the total but shows up in both product columns, so 163 + 87 can exceed a total of 217. Nothing is double-counted.

Why Go-Lives bars may not match the headline
Bars show full weeks or months. If your period starts mid-month, the first bar still shows the whole month, so the bars can add past the headline. The headline is always the exact count for your dates. Trust the headline for totals, the bars for shape.

Why PO to Go-Live has gaps
That chart only plots days when something went live. Quiet days aren't drawn, so points sit closer together than the calendar would suggest when you zoom in to daily detail.

Implementations that can't be counted
An implementation without a create date can't be placed on the timeline, so it's left out of the open counts. The speed average skips implementations with no Purchase Order date, or where the dates run backwards. Real HubSpot exports set these automatically, so this mostly matters for hand-edited files.

Your data
Everything runs in your browser. Nothing uploads anywhere. Back up saves your history to a file, Restore loads it back, and Reset wipes this browser clean. Back up before you reset.
Import your HubSpot export to begin
What to export (records, not a report):
1. In HubSpot: CRM → Implementations → All records → Export.
2. Format CSV · language English · include All properties on records.
3. Include open and closed implementations (no filters).
4. Name it like implementations_2026-07-01.csv (the date tells the app which month it counts as).
5. Drop it here or click Import. Re-importing later just refreshes the data.
Required fields and assumptions
Must be present (exact HubSpot names): Record ID · Implementation Name · Implementation Type · Implementation pipeline stage · Time in current stage · Date entered current stage · PO Date · Implementation Live/Complete · Duration between PO date and Live/Complete Date · Object create date/time, plus the per-stage Date entered/exited "[stage] (Implementation Pipeline)" columns (included automatically with "All properties").

Assumptions the math relies on: one row per implementation · durations are HubSpot's raw milliseconds · dates are YYYY-MM-DD or MM/DD/YYYY · Implementation Type values contain "CAREpoint" / "e-Bridge" (both = counted in both) · the Live/Complete date means the customer went live · moving to the Live/Complete stage means the record left the pipeline.
