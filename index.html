How to use

HubSpot shows today. This shows the trend over time. It opens to an import screen with optional sample dashboards, so you can look around before importing anything real.

Do this each month
  1. Export from HubSpot.
    • In HubSpot's top menu, click CRM, then Implementations.
    • Set the view to All Implementations. You want every record, both open and completed, not a filtered subset. A filtered view leaves records out and the numbers come out wrong.
    • Click Export (top right), choose CSV, and download the file.
    • Name it with the last day of the month, like implementations_2026-06-30.csv.
  2. Import the CSV. Click Import CSV and choose the file you just saved. Re-importing a month replaces it, so the picture builds over time.
  3. Read the three top cards. Open is how many implementations are active, and whether that rose or fell since last month. Where it's stuck is the total amount of time work is sitting past its normal pace, added up across everything (a total amount of time, not a count of implementations), plus the stages causing most of it. Delivery is the typical number of days from PO to Go-Live, and how many went live.
  4. Check where work is stuck. Time in Stage lists stages worst first. "Days overdue" is the extra time work has spent past that stage's normal benchmark, added together across the items in that stage, not a number of implementations. Click a stage to chart it and see who's waiting. Stage Breakdown grades every stage at once.
  5. Review Go-Lives / PO to Go-Live. The typical wait to go live and what shipped this period. The "went live, not closed out" link lists records still open in HubSpot after their go-live date.
  6. Save or share. Open the ⋮ menu to export a PNG or PDF, or share a snapshot.
  7. Back up history. ⋮ menu → Back up saves your history to a file; Restore loads it back; Reset wipes this browser. Back up before you reset, and keep the file in GD's Drive.
Required fields & assumptions
  • Required columns (exact HubSpot names): Record ID · Implementation Name · Implementation Type · Implementation pipeline stage · Time in current stage · Date entered current stage · PO Date · Implementation Live/Complete · Duration between PO date and Live/Complete Date · Object create date/time. Also used when present: Estimated Go-Live Date and the per-stage "Date entered/exited … (Implementation Pipeline)" columns.
  • Timeframes: pick a preset or exact dates; opens on the last full month. Product buttons (CAREpoint / e-Bridge) narrow most of the dashboard; the Open card currently stays on the combined total. Charts compare your range to the equal-length period right before it. Stage verdicts and overdue describe the latest import (30-day flow window) and don't move when you change the date range.
  • Colors: green improving, red worsening, amber watch. The dotted line marks where the visible period started; below it is better than the start, above is worse.
  • Counting: an implementation can be both CAREpoint and e-Bridge, counted once in the total but shown in both product columns, so product columns can add past the total. Nothing is double-counted. No create date leaves a record out of open counts; a missing or backward PO date is skipped by the speed median.
  • Charts: Ctrl/⌘ + scroll to zoom, drag to pan, double-click to reset. On Time in Stage, the ‹ handle hides the stage list for full width.
  • Your data runs in your browser on DuckDB (the rules live in engine.sql). Nothing uploads anywhere.

Required fields & assumptions
CRM → Implementations → All Implementations view, every filter cleared (open and completed both matter). Export → CSV; the standard 45-column view has everything. Name it implementations_YYYY-MM-DD.csv.

Must include (exact HubSpot names): Record ID · Implementation Name · Implementation Type · Implementation pipeline stage · Time in current stage · Date entered current stage · PO Date · Implementation Live/Complete · Duration between PO date and Live/Complete Date · Object create date/time, plus the per-stage Date entered/exited "[stage] (Implementation Pipeline)" columns and Estimated Go-Live Date.

Assumptions: one row per implementation · durations are raw milliseconds · dates are YYYY-MM-DD or MM/DD/YYYY · Type contains "CAREpoint" / "e-Bridge" (both = counted in both) · Live/Complete date = went live · Live/Complete stage = left the pipeline.
