How to use

Jump to: What this is · Monthly routine · The export · Reading it · Colors · Timeframes · Charts · Counting quirks · Your data
What this is
  • HubSpot shows today; this shows the trend over time.
  • Import your monthly export to track open implementations, stage times, and delivery speed.
  • Opens with sample data — your first real import replaces it.
Monthly routine
  • Export all Implementation records from HubSpot.
  • Import the CSV here.
  • The dashboard remembers each month, so the picture builds over time.
Getting the export right
  • CRM → Implementations → All Implementations view, every filter cleared (open and completed both matter).
  • Export → CSV, properties in your view's columns (the standard 45-column view has everything).
  • Name it implementations_YYYY-MM-DD.csv.
  • Must include (exact HubSpot names): Record ID · Implementation Name · Implementation Type · Implementation pipeline stage · Time in current stage · Date entered current stage · PO Date · Implementation Live/Complete · Duration between PO date and Live/Complete Date · Object create date/time. Also used when present: Estimated Go-Live Date and the per-stage "Date entered/exited … (Implementation Pipeline)" columns.
Reading the dashboard
  • Top cards — trend (Rising / Steady / Falling) vs the period before, for open work, each product, and Go-Lives.
  • Open Pipeline Trend — whether open work is piling up or clearing.
  • Time in Stage — where work is stuck. Overdue = days open work sits past a stage's benchmark; ranked biggest bottleneck first. Click a stage to chart it and see who's waiting.
  • Stage Breakdown — every stage graded at once, with sortable columns.
  • PO to Go-Live / Go-Lives — median wait to go live, and what shipped (plus a "went live but not closed out" link).
What the colors mean
  • Green = improving, red = worsening, amber = watch.
  • The dotted line marks where the visible period started: below it is better than the start, above is worse.
  • Go-Lives bars are neutral (a count); the change pill greens when more shipped than the period before.
Timeframes and comparisons
  • Pick a preset or set exact dates; opens on the last full month. Product buttons filter the whole dashboard.
  • Charts and trend pills recalc for your range vs the equal-length period right before it.
  • Stage verdicts and Overdue describe the latest import (30-day flow window) and don't move when you change the date range.
Exploring a chart
  • Ctrl/⌘ + scroll to zoom, drag to pan, double-click to reset.
  • On Time in Stage, the ‹ handle hides the stage list so the chart gets full width.
Counting quirks
  • An implementation can be both CAREpoint and e-Bridge — counted once in the total but shown in both product columns, so product columns can add past the total. Nothing is double-counted.
  • Go-Lives bars show full weeks/months, so they can add past the headline. Trust the headline for totals, bars for shape.
  • PO to Go-Live only plots days something went live; quiet days aren't drawn.
  • No create date → left out of open counts. No PO date (or backwards dates) → skipped by the speed median. Real exports set these automatically.
Your data
  • Everything runs in your browser on DuckDB (the rules live in engine.sql). Nothing uploads anywhere.
  • Back up saves your history to a file, Restore loads it back, Reset wipes this browser. Back up before you reset.
Import your HubSpot export to begin
1. In HubSpot: CRM → Implementations → All Implementations view, every filter cleared (open and completed both matter).
2. Export → CSV, including the properties in your view's columns.
3. Name it implementations_2026-07-01.csv and drop it here.
Required fields and assumptions
Your standard 45-column view has everything ("All properties" also works, just bigger). Re-importing later just refreshes the data.

Must be present (exact HubSpot names): Record ID · Implementation Name · Implementation Type · Implementation pipeline stage · Time in current stage · Date entered current stage · PO Date · Implementation Live/Complete · Duration between PO date and Live/Complete Date · Object create date/time, plus the per-stage Date entered/exited "[stage] (Implementation Pipeline)" columns and Estimated Go-Live Date (all part of the standard view). Verified against the live HubSpot pipeline and three real exports on Jul 3, 2026.

Assumptions the math relies on: one row per implementation · durations are HubSpot's raw milliseconds · dates are YYYY-MM-DD or MM/DD/YYYY · Implementation Type values contain "CAREpoint" / "e-Bridge" (both = counted in both) · the Live/Complete date means the customer went live · moving to the Live/Complete stage means the record left the pipeline.
